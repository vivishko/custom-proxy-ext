# Bug-01. Enable only on this page disables proxy

## Context / why
Текущая кнопка "Enable only on this page" должна оставлять прокси активным только для текущего домена, а для остальных — выключать. Сейчас после нажатия показывается статус "Proxy temporarily disabled for www.linkedin.com", что выглядит как отключение именно на текущем домене и конфликтует с ожидаемым поведением.

## Goals
- Исправить поведение кнопки так, чтобы прокси оставался включенным для текущего домена.
- Обновить статус/текст объяснения, чтобы он совпадал с фактическим режимом.

## Non-goals
- Редизайн UI статуса прокси.
- Новые режимы переключения помимо "global" и "only on this page".

## User scenarios
- Пользователь включает прокси и нажимает "Enable only on this page" — прокси активен только для текущего домена.
- Пользователь видит статус, который явно указывает "Enabled for текущий домен, disabled elsewhere".

## Requirements
- Must: При выборе "Enable only on this page" текущий домен получает активный прокси.
- Must: Для остальных доменов прокси не применяется.
- Must: Строка статуса не должна говорить, что прокси отключен для текущего домена.
- Should: Текст статуса явно показывает домен, для которого прокси активен.
- Could: Показать краткое пояснение, как вернуть глобальный режим.

## Risks / questions
- Где именно формируется логика “temporarily disabled”: в правиле, UI или сервисе.
- Нужно ли учитывать текущие правила пользователя (например, если уже есть правило для домена).

## Plan (steps)
- Найти место формирования правила для режима "only on this page".
- Проверить соответствие логики (инверсия или неверное сообщение).
- Исправить режим и статусный текст.
- Проверить на нескольких доменах.
